# 单词卡片 (WordCards) - PRD 文档

## 1. 产品概述

- **产品名称**: WordCards - 单词卡片记忆
- **产品类型**: 工具类 App
- **核心功能**: 用户可以创建自定义单词卡片，进行间隔重复记忆
- **目标用户**: 学生、语言学习者、备考人群
- **核心价值**: 随时随地创建卡片、复习记忆、巩固知识点

## 2. 功能清单

### 2.1 卡片管理
- [ ] 创建卡片（正面单词/问题，背面解释/答案）
- [ ] 编辑卡片
- [ ] 删除卡片
- [ ] 卡片分类（创建不同的卡组/ deck）
- [ ] 导入/导出卡组（JSON格式）

### 2.2 学习模式
- [ ] 翻转卡片（点击切换正面/背面）
- [ ] 认识/不认识 标记
- [ ] 随机顺序复习
- [ ] 按顺序复习
- [ ] 复习进度显示

### 2.3 学习统计
- [ ] 今日学习数量
- [ ] 累计学习卡片数
- [ ] 学习时长统计

### 2.4 设置
- [ ] 深色/浅色主题切换
- [ ] 语言选择（英语/中文/日语/西班牙语/法语/德语，界面UI语言）
- [ ] 数据本地存储（LocalStorage）

## 3. 界面设计

### 3.1 主页
- 卡组列表（显示名称 + 卡片数量 + 进度）
- "创建卡组" 按钮
- "学习" 按钮（进入学习模式）
- 设置入口

### 3.2 卡组详情
- 卡片列表预览
- 添加卡片入口
- 编辑卡组名称
- 开始学习按钮

### 3.3 卡片创建/编辑
- 正面输入框
- 背面输入框
- 保存/取消按钮

### 3.4 学习模式
- 大面积卡片展示（点击翻转）
- 底部：认识 ✓ / 不认识 ✗ 按钮
- 进度指示器
- 退出按钮

### 3.5 设置页
- 主题切换
- 语言选择
- 关于/版本信息

## 4. 技术方案

### 4.1 技术栈
- **框架**: Vanilla JS + HTML + CSS（轻量、加载快）
- **打包**: Capacitor（生成 Android APK）
- **存储**: LocalStorage（无需后端）
- **图标**: 内联 SVG 或 emoji

### 4.2 目录结构
```
word-cards/
├── index.html      # 主页面
├── css/
│   └── style.css   # 样式
├── js/
│   └── app.js      # 逻辑
├── assets/
│   └── icon.png    # App图标
└── capacitor.config.json
```

### 4.3 数据模型
```javascript
// 卡组
{
  id: "uuid",
  name: "CET-4 词汇",
  cards: [
    {
      id: "uuid",
      front: "abandon",
      back: "放弃；遗弃"
    }
  ],
  createdAt: timestamp,
  lastStudied: timestamp
}
```

## 5. 定价策略

- **买断制**: $0.99 - $1.99（首发 $0.99）
- **无内购**: 简洁无广告
- **免费版**: 无限制使用（仅作为引流，后期可考虑限制卡组数量）

## 6. 上架策略

### 6.1 Google Play
- **隐私政策**: 需提供简单隐私政策（说明数据仅本地存储）
- **年龄分级**: 所有人
- **定价**: 设置为付费

### 6.2 风险控制
- 功能单一，审核风险低
- 无第三方SDK，隐私合规
- 不使用敏感权限

## 7. 开发计划

| 阶段 | 任务 | 预计工时 |
|------|------|----------|
| 1 | 项目搭建 + 基础框架 | 2h |
| 2 | 卡片 CRUD 功能 | 3h |
| 3 | 学习模式 + 翻转动画 | 2h |
| 4 | 统计功能 | 1h |
| 5 | 主题 + 多语言 | 1h |
| 6 | 打包 + 测试 | 2h |
| **总计** | | **~11h** |

## 8. 美术设计

- **风格**: 简洁清新，卡片式设计
- **配色**: 
  - 主色: #4A90D9（蓝色，理性/学习）
  - 背景: #F5F7FA（浅灰白）
  - 卡片: #FFFFFF
  - 文字: #333333
- **字体**: 系统默认字体
- **图标**: 使用 emoji 或简单 SVG
- **动画**: 卡片翻转效果（3D transform）

## 9. 注意事项

- ⚠️ 避免功能过于复杂，保持"一个app解决一个问题"
- ⚠️ 数据必须支持导出，防止用户数据丢失
- ⚠️ 首次启动可以展示简短引导页
